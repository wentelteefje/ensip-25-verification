Understanding Interoperable Addresses

ERC-7930 and ERC-7828 introduce a standardized way to represent blockchain addresses that work across multiple chains. Instead of managing different addresses for different networks, you can use a single human-readable identifier.

The format is simple: name@chain. For example, vitalik.eth@base resolves to Vitalik's address on the Base network. Try it out in the interactive demo below:

This interoperable format makes cross-chain interactions more intuitive and reduces the risk of sending assets to the wrong address on the wrong chain.

How It Works

The demo above resolves ENS names to their corresponding addresses and combines them with chain identifiers (CAIP-2 format). The system looks up the ENS name, retrieves the Ethereum address, and presents it alongside the chain-specific identifier.
